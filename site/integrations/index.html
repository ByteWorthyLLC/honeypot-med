Integrations

Make Honeypot Med fit existing eval and CI workflows.

The project now exports formats that other tools already understand: GitHub Actions, SARIF, OpenTelemetry-style logs, promptfoo, Inspect AI, legacy OpenAI Evals, canonical JSONL, Markdown, SVG badges, and standalone HTML reports.

Portable export 
python app.py export --pack claims --format all --outdir reports/export

Use --format sarif, --format otel, --format markdown, --format badge, or --format eval-kit for focused artifacts.

GitHub Action

Run challenge mode in CI.

The root action.yml installs the project, runs challenge mode, uploads artifacts, and can upload SARIF to GitHub Code Scanning.

Workflow snippet

- uses: ByteWorthyLLC/honeypot-med@main
  with:
    pack: healthcare-challenge
    output-dir: honeypot-med-report
    fail-under: 70
    upload-artifact: true
    upload-sarif: true
Export formats

The artifact map.

Each format exists because a different audience needs to consume the same result.

HTML

Human-readable proof page for buyers, leadership, screenshots, and public galleries.

SARIF

Static-analysis format that can be uploaded into GitHub Code Scanning.

OpenTelemetry logs

Operational log payload that maps report events into telemetry-style records.

Eval kit

Offline adapters for promptfoo, Inspect AI, legacy OpenAI Evals, and canonical JSONL workflows.

JSON

Stable machine-readable report for custom dashboards and downstream eval systems.

Markdown

PR comments, release notes, docs, and simple diffs.

SVG badge

README, docs, and launch posts that show the score without opening the report.

Eval kit

Move traps into tools teams already use.

The default generated promptfoo config uses the echo provider, so adapter generation and assertion checks can stay offline until a team intentionally swaps in a live provider.

Offline adapter command

python app.py eval-kit --pack healthcare-challenge --outdir reports/eval-kit --engine-mode local --no-allow-network
